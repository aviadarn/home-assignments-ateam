Core Technical Approach

5-Stage Filtering Funnel

① EMBED all-MiniLM-L6-v2 · 384-dim · ~2,000 posts/sec CPU all posts
② NEAR-DEDUP cosine sim > 0.97 → drop reshares & paraphrases
③ VELOCITY engagement_growth_rate > 1.5 over 6hr → trending tag
④ FILTER cosine > 0.55 → ~90% dropped ~50 posts
⑤ CLASSIFY 12-feature logistic · threshold 0.65 ~20 posts
⑥ LLM SCORE Bedrock Haiku · top 20 ≤10 alerts
WHY LOGISTIC REGRESSION?
Interpretable, fast, updatable weekly from ~200 examples. Neural model needs thousands per brand. Cold-start problem.

B — LEARNING TARGET (Zephyr Energy)
CORE (unchanged): "energy drink" 1.000 "focus" 1.000 "productivity" 1.000 EXPANSION (learned): "afternoon slump" 0.820 ← new "deep work" 0.710 ← new "L-theanine" 0.650 ← new "nootropics" 0.410 → ("energy drink" OR focus OR productivity OR "deep work" OR "afternoon slump") AND NOT ("extreme sports" OR "weight loss")
C — SCORING FORMULA
score = click_rate × action_rate × 10 + log(freq + 1)
click_rate = clicked / surfaced
action_rate = actioned / clicked
Promote if: score ≥ 0.5 AND freq ≥ 3
Decay: new = 0.9 × old + 0.1 × current
Prune: weight < 0.3 for 3+ weeks
D — EVALUATION
Primary Metric CTR = actions / delivered
12% → 18% target in 8 weeks
68.4% → 89.5% code demo (+31% rel.)
Guardrail: CTR < 8% any brand → reset to guidelines baseline + human review
Loops 2 & 3 · Bi-weekly & Monthly

Calibrating Filters & Improving Reasoning

LOOP 2 · THRESHOLD CALIBRATION
"How aggressively to filter"
Target: per-brand classifier_threshold (default 0.65, range [0.45, 0.85])
1. Collect (score, label) pairs — clicked=1, dismissed=0
2. Sweep thresholds 0.40→0.90 in 0.05 steps
3. Maximize F-0.5 score
4. Dampen: new = old + clip(optimal−old, −0.05, +0.05)
WHY F-0.5?
"Brand manager attention is scarce. An irrelevant alert trains them to ignore the dashboard. A missed trend is recoverable — they'll catch it next week. So I penalize false positives 2× more than false negatives."
Precision@10 baseline 2.5 / 10
Target (12 weeks) 4 / 10
LOOP 3 · PROMPT OPTIMIZATION
"How to reason about guidelines"
Target: s3://prompt-store/{"{"}brand_id{"}"}/relevance_prompt_v{"{"}N{"}"}.txt
1. Collect failure cases (wrong suppression / dismiss <2s)
2. LLM-assisted pattern extraction → 2-3 error patterns
3. Construct candidate prompt (new version)
4. A/B test: 20% traffic for 1 week
5. Promote if CTR +5% AND no suppression rate increase
WHY MONTHLY?
"Prompt changes affect all 40 brands simultaneously. Monthly cadence with mandatory A/B window gives safety margin. The other loops change a single number per brand."
Suppression precision 75%
Target (6 months) 90%
measured by sampling 50 suppressed alerts/month
